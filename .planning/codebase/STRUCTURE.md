## Key File Locations

**Entry Points:**
- `server.ts`: Node/Express process entry; starts the API server, registers middleware, and serves the SPA.
- `index.html`: Browser HTML entry; defines `#root` and loads `/src/main.tsx`.
- `src/main.tsx`: React entry; renders `App` and imports `src/index.css`.
- `src/App.tsx`: Primary UI entry; owns the form, client request, and report rendering.

**Configuration:**
- `package.json`: Defines `dev`, `start`, `build`, `preview`, `clean`, and `lint` scripts.
- `package-lock.json`: Locks npm dependency versions.
- `vite.config.ts`: Configures React/Tailwind plugins, path alias, HMR behavior, and `process.env.GEMINI_API_KEY` define.
- `tsconfig.json`: Configures TypeScript target, JSX mode, module resolution, and `@/*` alias.
- `.gitignore`: Ignores `node_modules/`, `build/`, `dist/`, `coverage/`, logs, `.env*`, `.playwright-cli/`, and `output/`.
- `.env.example`: Public environment template; note existence only for `.env.local`.
- `playwright-cli.json`: Browser verification tooling configuration.
- `metadata.json`: App name, description, and frame permissions metadata.

**Core Logic:**
- `server.ts`: Request lifecycle, PDF extraction, alias mention extraction, provider routing, fallback report generation, hydration, error handling.
- `src/lib/analysis.ts`: Sequence normalization, FASTA header extraction, local heuristic scanning, artifact hashing, provenance root construction.
- `src/lib/protein-resolver.ts`: FASTA/header parsing, known public protein fallback, UniProt lookup, identity/cross-reference assembly.
- `src/lib/schema.ts`: `BioVizReport` runtime schema, TypeScript type, and JSON Schema for model providers.

**UI Logic:**
- `src/App.tsx`: Form submission, loading/error/report state, protein track visualization, canonical identity panel, provenance panel, region cards, paper mention cards, database link list.
- `src/index.css`: Tailwind import.

**Generated Outputs:**
- `output/playwright/glp1-demo/`: Demo screenshots, videos, PDF input, and browser recording script.
- `output/presentation/`: Presentation slides, audio, video segments, manifests, final videos, and generation script.
- `.playwright-cli/`: Local verification run output.

**Testing:**
- Not detected. There are no `*.test.*` or `*.spec.*` files in the scanned project root or `src/`.
- `package.json` exposes `npm run lint` as `tsc --noEmit`; no test script is defined.

## Naming Conventions

**Files:**
- Use lowercase kebab-case for shared library files: `src/lib/protein-resolver.ts`.
- Use conventional React root names for UI files: `src/App.tsx`, `src/main.tsx`.
- Use root-level config filenames expected by tools: `vite.config.ts`, `tsconfig.json`, `package.json`.
- Use uppercase planning document names under `.planning/codebase/`: `.planning/codebase/ARCHITECTURE.md`, `.planning/codebase/STRUCTURE.md`.

**Directories:**
- Use `src/` for application source.
- Use `src/lib/` for non-UI shared TypeScript modules.
- Use `.planning/` for GSD workflow artifacts.
- Use `output/` for generated demo/presentation artifacts.
- Use `.playwright-cli/` for generated browser automation artifacts.

**Exports:**
- Export shared primitives from `src/lib/*.ts` by name: `normalizeSequence`, `runLocalHeuristics`, `ProvenanceGraph`, `resolveProtein`, `BioVizReportSchema`.
- Export the React root component as the default export from `src/App.tsx`.

## Module Boundaries

**Keep UI Code In `src/App.tsx`:**
- Put form fields, browser `fetch`, component state, and report rendering in `src/App.tsx`.
- Use `BioVizReport` from `src/lib/schema.ts` for report typing.
- Do not put provider calls, PDF extraction, or protein resolver network calls in `src/App.tsx`.

**Keep HTTP/API Orchestration In `server.ts`:**
- Put Express routes, middleware, provider routing, environment reads, PDF handling, and response assembly in `server.ts`.
- Use `src/lib/analysis.ts`, `src/lib/protein-resolver.ts`, and `src/lib/schema.ts` as imported helpers.
- Keep `/api/analyze` as the current one-shot analysis endpoint unless a new workflow requires a separate route.

**Keep Deterministic Analysis In `src/lib/analysis.ts`:**
- Put pure or mostly deterministic sequence utilities in `src/lib/analysis.ts`.
- Use `normalizeSequence` and `extractFastaHeader` before calling downstream analysis or model providers.
- Use `ProvenanceGraph` for per-request artifact hashing when adding new analysis steps.

**Keep Identity Enrichment In `src/lib/protein-resolver.ts`:**
- Put FASTA/header parsing, alias handling, sequence-based known public fallback, and UniProt response mapping in `src/lib/protein-resolver.ts`.
- Return conservative empty/fallback identity objects when resolution fails.
- Keep resolver outputs compatible with `canonicalProtein`, `aliases`, and `databaseLinks` in `src/lib/schema.ts`.

**Keep Report Shape In `src/lib/schema.ts`:**
- Update `BioVizReportSchema`, `BioVizReport`, and `bioVizJsonSchema` together when changing the report contract.
- Keep the schema compatible with both model-provider structured output and client rendering in `src/App.tsx`.
- Treat `src/lib/schema.ts` as the boundary contract between backend synthesis and frontend display.

**Keep Generated Artifacts Out Of Runtime Imports:**
- Do not import from `output/` or `.playwright-cli/` in application code.
- Treat `output/` as reproducible demo/presentation output, not a source module directory.

## Where to Add New Code

**New API Endpoint:**
- Primary code: `server.ts`
- Shared schemas: `src/lib/schema.ts`
- Shared deterministic helpers: `src/lib/analysis.ts`
- UI caller: `src/App.tsx`

**New Analysis Step:**
- Primary code: `src/lib/analysis.ts` when the step is deterministic and sequence-local.
- Request orchestration: `server.ts` to call the step, record provenance, and merge output into the response.
- Contract changes: `src/lib/schema.ts` when the result appears in `BioVizReport`.

**New Protein Identity Or Database Enrichment:**
- Primary code: `src/lib/protein-resolver.ts`
- Request orchestration: `server.ts`
- Contract changes: `src/lib/schema.ts` if new identity fields or database link shapes are exposed.

**New Model Provider:**
- Primary code: `server.ts`
- Add a provider-specific adapter function near `runOpenAIModel` and `runLocalModel`.
- Validate provider output with `BioVizReportSchema` from `src/lib/schema.ts`.
- Add any required environment names to `.env.example`.

**New Report Field:**
- Schema: `src/lib/schema.ts`
- Server population: `server.ts`
- Client rendering: `src/App.tsx`
- Provider JSON Schema: update `bioVizJsonSchema` in `src/lib/schema.ts` in the same change as `BioVizReportSchema`.

**New UI Section:**
- Implementation: `src/App.tsx`
- Styling: Prefer Tailwind classes directly in `src/App.tsx`; global CSS currently only imports Tailwind in `src/index.css`.
- Type source: Use `BioVizReport` from `src/lib/schema.ts`.

**New Utility:**
- Shared analysis utility: `src/lib/analysis.ts`.
- Protein/identity utility: `src/lib/protein-resolver.ts`.
- Server-only utility: keep inside `server.ts` unless it becomes reusable or crosses module boundaries.

**New Generated Demo Or Presentation Asset:**
- Browser/demo artifacts: `output/playwright/`.
- Presentation artifacts: `output/presentation/`.
- Local verification run artifacts: `.playwright-cli/`.
- Keep generated files ignored unless the project intentionally changes `.gitignore`.
